02 — Chi Tiết Từng Bước

Prompt & hướng dẫn

Mô tả đầy đủ, prompt mẫu, và lưu ý thực tế cho từng bước.

01
Nguồn dữ liệu

File lời thoại cuộc họp

Không cần file hoàn hảo. Transcript thô, ghi chú lộn xộn, hay copy từ phần mềm ghi âm đều được. Claude sẽ lo phần làm sạch.

Yêu cầu tối thiểu

Chủ đề buổi họp — tên người trình bày — nội dung chính (framework, kiến thức, ví dụ) — kết luận hoặc bài học rút ra.

Định dạng chấp nhận
  • .txt — khuyến nghị, Claude đọc nhanh và chính xác nhất
  • .docx — upload thẳng vào Claude
  • Copy/paste nội dung thẳng vào hộp chat
Mẹo

Ghi lại tên người nói nếu có nhiều diễn giả. Claude sẽ tổ chức nội dung tốt hơn khi biết ai đang nói gì.

02
Xử lý AI

Claude đọc & cấu trúc nội dung

Đưa nội dung cho Claude phân tích. Duyệt cấu trúc trước khi tạo HTML — sửa ở đây dễ hơn sửa sau.

Prompt mẫu — Bước 02 
Đây là nội dung lời thoại từ buổi [tên buổi họp] ngày [ngày].

[DÁN NỘI DUNG VÀO ĐÂY]

Hãy:
1. Đọc và hiểu toàn bộ nội dung
2. Xác định các chủ đề chính, framework, và bài học cốt lõi
3. Tổ chức lại theo cấu trúc: vấn đề → giải pháp → ứng dụng thực tế
4. Tóm tắt từng phần đủ rõ để làm tài liệu web
5. Báo cáo lại cấu trúc dự kiến (danh sách section và nội dung chính)
   để tôi duyệt trước khi tạo HTML
Tại sao cần bước này?

Dành 2 phút duyệt output của Claude ở đây, tiết kiệm 20 phút chỉnh sửa HTML sau. Đảm bảo anh và Claude đồng ý về cấu trúc trước khi tạo file.

03
Xử lý AI

Claude xuất file index.html

Sau khi duyệt cấu trúc, dùng prompt này để Claude tạo file HTML hoàn chỉnh theo brand Pharmesthetic.

Prompt mẫu — Bước 03 
Dựa trên nội dung và cấu trúc đã thống nhất, hãy tạo file index.html
theo đúng thông số sau:

THIẾT KẾ:
- Nền: #13182A (navy đậm)
- Nhấn: #B89B6C (champagne gold)
- Chữ: #EDE8DC (cream)
- Heading: font Cormorant Garamond (Google Fonts)
- Body: system-sans
- Không dùng icon, không emoji, không viền trang trí

BỐ CỤC:
- Hero section: tên buổi, ngày, người trình bày
- Navigation bar sticky với scroll-spy
- Từng section ứng với từng chủ đề chính
- Footer tối giản

KỸ THUẬT:
- Một file duy nhất, tự chứa (CSS và JS nhúng trong file)
- Không phụ thuộc external server ngoài Google Fonts
- Responsive, hoạt động tốt trên điện thoại
- Mở thẳng trên trình duyệt là chạy được (không cần localhost)
- Thêm hiệu ứng scroll-reveal cho từng section

Tạo file và cho tôi download.
Sau khi nhận file

Mở thẳng bằng trình duyệt để kiểm tra trước khi upload GitHub. Cần có internet mới load được font Cormorant Garamond từ Google Fonts.

04
Triển khai

Upload lên GitHub repo

Không cần dùng Claude. Thực hiện thủ công trên GitHub.com — khoảng 2 phút.

  1. Đăng nhập github.com
  2. Bấm dấu + góc trên phải → New repository
  3. Đặt tên repo (ví dụ: mentor-foundation-ss4) → chọn Public → Create repository
  4. Bấm Add file → Upload files
  5. Kéo thả hoặc chọn file index.html
  6. Gõ commit message (ví dụ: Add session 4 web)
  7. Bấm Commit changes
Bắt buộc

Tên file phải là index.html — chữ thường, không dấu cách, không đuôi khác. GitHub Pages chỉ tự nhận đúng tên này.

05
Triển khai

Bật GitHub Pages

Vẫn không cần Claude. Thực hiện trong phần Settings của repo — 1 phút.

  1. Vào tab Settings trên thanh menu của repo
  2. Bấm Pages trong menu trái
  3. Source: chọn Deploy from a branch
  4. Branch: chọn main → thư mục / (root)
  5. Bấm Save
  6. Chờ 1–3 phút → Reload trang Settings → Pages
  7. Thấy link màu xanh là deploy xong
Lỗi thường gặp
Trang trắng hoặc hiện danh sách file

File chưa đặt đúng tên index.html. Vào repo, bấm vào file → icon bút chì → đổi tên → Commit changes.

Chưa hiện dù đã deploy

Mở tab ẩn danh (Ctrl+Shift+N) rồi dán link vào test. Browser cache cũ không ảnh hưởng tab ẩn danh. Nếu vẫn chưa ra, chờ thêm 2–3 phút.

Checklist trước khi gửi
  • Mở trên điện thoại — test mobile layout
  • Test cả Chrome lẫn Safari
  • Bấm thử từng mục trong navigation bar
  • Font Cormorant Garamond hiển thị đúng chưa (cần internet)
Cập nhật sau này

Chỉnh file index.html → upload đè lên GitHub repo → GitHub Pages tự cập nhật sau 1–2 phút. Không cần làm lại từ đầu.
